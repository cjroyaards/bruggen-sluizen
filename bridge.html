NMEA-BRUG ← terug naar de kaart

Boordinstrumenten koppelen via de NMEA-brug

Heeft je gateway alleen een TCP- of UDP-poort (zoals de Digital Yacht NavLink2/WLN10 of een YDWG zonder Web Gauges)? Browsers kunnen daar niet rechtstreeks bij. Deze mini-brug — één klein programma, niets te installeren behalve Python — leest je gateway uit en geeft de data door als WebSocket waar OpenPilot wél bij kan.

1 · Download (zet beide in dezelfde map, bijv. Downloads)

nmea-bridge.py — de brug starter voor Mac starter voor Windows

2 · Start

Mac: dubbelklik nmea-brug.command. Blokkeert macOS het ("niet-geverifieerde ontwikkelaar")? Rechtsklik → Open → Open. Windows: installeer eenmalig Python (vink "Add to PATH" aan) en dubbelklik nmea-brug.bat.

Vul het IP en de poort van je gateway in (bijv. 192.168.0.80 en 1456). Liever de terminal? Dan direct:

python3 nmea-bridge.py --tcp 192.168.0.80:1456
python3 nmea-bridge.py --udp 2000

Gelukt als je ziet: [brug] gateway verbonden ✓.

3 · Verbind in OpenPilot

Zeekaartmodus → Plotter → NMEA → vul in ws://localhost:8100 → verbind. SOG/COG, diepte en wind komen dan van je boordinstrumenten.

Let op: ws://localhost:8100 werkt alleen als de brug op dezelfde computer draait als je browser (Chrome, Edge of Firefox). Verbinden vanaf een ander apparaat (bijv. iPad → laptop-brug) kan pas als de site via http bereikbaar is — dat komt met het eigen domein. Je computer en de gateway moeten op hetzelfde (boot)netwerk zitten.

Stoppen: Ctrl-C (of het venster sluiten). De brug herstelt de gateway-verbinding automatisch en kan meerdere schermen tegelijk bedienen. Indicatief — niet voor navigatiedoeleinden.
